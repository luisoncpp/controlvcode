# PromptBuilder – Architecture

## 🧠 Purpose
`PromptBuilder` allows the user to compose complex prompts for an LLM by combining three sources:
1. The feedback XML (`<execution_results>`) generated by command execution.
2. A free-text message written by the user.
3. Project files attached via interactive search using `@`.

Everything is encapsulated in `src/components/PromptBuilder/`, with a single entry point `index.tsx` that exposes no internal dependencies to the rest of the application.

---

## 🗂️ File structure

```
src/components/PromptBuilder/
  index.tsx              ← Facade component (only export)
  PromptBuilder.ts       ← Logic and state class
  PromptBuilderUI.tsx    ← Main visual component
  FileSearchDropdown.tsx ← File autocomplete dropdown
  types.ts               ← Local types (AttachedFile)
```

---

## 📄 File responsibilities

### `index.tsx`
**Role**: Public entry point. Instantiates the `PromptBuilder` class and renders the UI.

```tsx
export function PromptBuilderComponent({ feedbackXml }: { feedbackXml: string }) {
  const builder = new PromptBuilder(feedbackXml);
  return <PromptBuilderUI builder={builder} />;
}
```

- Receives the feedback XML as a prop from `App.tsx`.
- Creates a new instance every time `feedbackXml` changes (Preact optimises re-renders). Could be memoised in the future.

### `PromptBuilder.ts`
**Role**: All state, search, attachment, and prompt-building logic. A pure class using Preact `signal`s for reactivity.

#### State (signals)
- `message`: User's free text.
- `attachedFiles`: List of attached files (`AttachedFile[]`).
- `searchResults`: Current file search results.
- `showDropdown`: Controls dropdown visibility.
- `searchQuery`: Substring after `@` used for searching.

#### Main methods

| Method | Description |
|--------|-------------|
| `onMessageChange(text)` | Updates `message`. If it contains `@` without a trailing space, activates debounced search (200ms). |
| `attachFile(path)` | Invokes `read_file_content`, adds the file to `attachedFiles`, and clears the search. |
| `removeFile(path)` | Removes an attachment by path. |
| `buildPrompt()` | Combines feedback + message + attachments into a final string. |
| `copyToClipboard()` | Copies the generated prompt to the clipboard. |

#### Debounced search logic
- Every time `@` is detected, the query is saved and a `setTimeout` of 200ms is triggered. If the user keeps typing, the previous timer is cancelled. This prevents excessive backend calls.
- Invokes the Tauri command `search_files` with the query.
- Results are assigned to `searchResults`, and `showDropdown` is activated if there is at least one result.

### `PromptBuilderUI.tsx`
**Role**: Main visual component for the composer. Renders:
- Header with title and "Generate & Copy" button.
- Attached file chips (each with a × button to remove).
- Textarea with placeholder and instructions.
- `FileSearchDropdown` positioned absolutely over the textarea.

**Local state**: `copied` (boolean) to show copy confirmation.

### `FileSearchDropdown.tsx`
**Role**: File search results dropdown. Positioned absolutely below the textarea (`bottom-full`). Each item shows a file icon and the path. On click, calls `builder.attachFile(path)`.

## 🔌 Backend interaction

`PromptBuilder` communicates exclusively with two Tauri commands:

### `search_files(query: String) -> Vec<String>`
- **Implementation**: In `lib.rs`, traverses `walkdir` from `project_root()`, filters ignored folders (`node_modules`, `.git`, etc.), and finds files whose names contain the query (case-insensitive). Limited to 20 results.
- **Usage**: `PromptBuilder.ts` → `onMessageChange` → debounce → `invoke("search_files", { query })`.

### `read_file_content(path: String) -> String`
- **Implementation**: Reads the UTF-8 content of the file at `project_root()/path`.
- **Usage**: `PromptBuilder.ts` → `attachFile` → `invoke("read_file_content", { path })`.

Both commands operate on `PROJECT_DIR` (static global) which is updated by the project folder selector.

## 🔄 Composition session lifecycle

```
1. [App.tsx] Renders <PromptBuilder feedbackXml={store.feedbackPrompt.value} />
2. [index.tsx] Creates new PromptBuilder instance with feedback XML.
3. [User] Types message in the textarea.
   a. If typing "@file":
      - PromptBuilder.onMessageChange detects the @.
      - Triggers debounced search to Tauri.
      - FileSearchDropdown shows results.
   b. If selecting a file:
      - PromptBuilder.attachFile(path) reads content via Tauri.
      - Adds visual chip and clears the @query from the message.
4. [User] Can attach multiple files or remove them.
5. [User] Clicks "Generate & Copy".
   a. PromptBuilder.buildPrompt() builds the combined string.
   b. PromptBuilder.copyToClipboard() copies to clipboard.
6. The prompt is ready to paste into the LLM.
```

## 🧪 Tests

`PromptBuilder.test.ts` contains 15 unit tests covering:
- State initialisation.
- `@` detection and search activation (with mocked `invoke`).
- Attaching files, preventing duplicates, handling errors.
- Removing attachments.
- Full prompt construction.
- Clipboard copy (simulated).

Tests are run with `npm test`.

---

## 📦 Isolation

- No file in `PromptBuilder/` imports from `ExecutionStore` or other components.
- The only external dependencies are `@tauri-apps/api/core` (for `invoke`) and `@preact/signals`.
- If migrating to another framework or changing the UI, only `PromptBuilderUI.tsx` and `FileSearchDropdown.tsx` need to be touched. The `PromptBuilder` class remains intact.

---

## 🗺️ Possible extensions

- **Keyboard shortcuts** in the dropdown (arrows + Enter).
- **Preview** of the generated prompt before copying.
- **Multiple projects** with persistent attachments per project.
- **Directory attachment** support.
